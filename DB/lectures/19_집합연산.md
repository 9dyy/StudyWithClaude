# 19강 — 집합 연산 (`UNION`, `INTERSECT`, `EXCEPT`)

> **목표:** 두 SELECT 결과를 **세로로** 합치고, 교집합/차집합을 구한다.
> **선수 지식:** 10~18강

---

## 1. JOIN과 집합 연산은 다르다

| 연산 | 방향 | 비유 |
|------|------|------|
| `JOIN` | **가로**로 합침 (컬럼이 늘어남) | 엑셀의 VLOOKUP |
| 집합 연산 | **세로**로 합침 (행이 늘어남) | 엑셀 시트 두 개 위아래로 붙이기 |

```text
JOIN:                            UNION:
A          B                     A          B
┌──┬──┐    ┌──┬──┐               ┌──┬──┐    ┌──┬──┐
│a1│a2│ JOIN │b1│b2│              │a1│a2│ UNION│b1│b2│
└──┴──┘    └──┴──┘               └──┴──┘    └──┴──┘
   ↓                                ↓
┌──┬──┬──┬──┐                     ┌──┬──┐
│a1│a2│b1│b2│                     │a1│a2│
└──┴──┴──┴──┘                     │b1│b2│
                                  └──┴──┘
```

수학의 집합처럼 **합집합(`UNION`) / 교집합(`INTERSECT`) / 차집합(`EXCEPT`)** 세 가지.

---

## 2. 공통 규칙 — 컬럼 수와 타입이 같아야

세 연산자 모두 다음 조건을 만족해야 한다.

1. **각 SELECT의 컬럼 개수가 같아야** 함
2. **대응되는 컬럼의 타입이 호환**돼야 함 (`INTEGER` ↔ `BIGINT`는 OK, `INTEGER` ↔ `TEXT`는 X)
3. 결과의 **컬럼 이름은 첫 번째 SELECT 것을 따른다**

```sql
-- ❌ 컬럼 수가 다름
SELECT name FROM users
UNION
SELECT name, level FROM admins;   -- ERROR: each UNION query must have the same number of columns

-- ❌ 타입 불일치
SELECT name FROM users
UNION
SELECT id FROM users;             -- TEXT vs BIGINT
```

---

## 3. `UNION` — 합집합 (중복 제거)

두 결과를 위아래로 쌓고, **중복된 행은 한 번만** 남긴다.

```sql
-- 길드 1번 또는 2번 소속 유저 이름 (중복 제거)
SELECT name FROM users WHERE guild_id = 1
UNION
SELECT name FROM users WHERE guild_id = 2;
```

이 예시는 `WHERE guild_id IN (1, 2)`로도 되지만, 집합 연산이 진가를 발휘하는 곳은 **다른 테이블이나 다른 형태의 결과를 합칠 때**다.

```sql
-- 알림 받을 모든 사람 = 관리자 + VIP 유저
SELECT email FROM admins
UNION
SELECT email FROM users WHERE is_vip = TRUE;
```

> 💡 `UNION`은 **중복 제거를 위해 정렬을 수행**한다. 큰 테이블에서는 느릴 수 있다.

---

## 4. `UNION ALL` — 중복 그대로

중복 제거 없이 **전부 쌓는다.** 보통 `UNION`보다 훨씬 빠르다.

```sql
SELECT name FROM users WHERE guild_id = 1
UNION ALL
SELECT name FROM users WHERE guild_id = 2;
```

같은 이름이 양쪽에 있어도 그대로 둘 다 나옴.

### `UNION` vs `UNION ALL` — 어느 걸?

| 상황 | 권장 |
|------|------|
| 중복이 절대 안 나올 게 확실하다 | `UNION ALL` (성능 ↑) |
| 중복을 어차피 제거하고 싶다 | `UNION` |
| 모르겠다 / 합치는 게 본 목적 | `UNION ALL` 먼저 고려 |

> 💡 **실무 팁:** 무심결에 `UNION`을 쓰는 사람이 많다. **중복 제거가 의미 없는 데이터**(예: PK가 서로 다른 테이블의 결과)는 `UNION ALL`이 정답. 성능 차이가 크다.

---

## 5. `INTERSECT` — 교집합

**양쪽 모두에 있는 행만** 남긴다.

```sql
-- 흑사자단(1)과 백호단(2) 양쪽에 동시에 있는 이름?
SELECT name FROM users WHERE guild_id = 1
INTERSECT
SELECT name FROM users WHERE guild_id = 2;
```

(한 유저가 두 길드에 동시 소속될 일이 없다면 비어 있겠지만, 개념상 동작은 그렇다.)

실용 예:

```sql
-- 결제도 했고, 게임도 플레이한 유저 (양쪽 활동 모두)
SELECT user_id FROM payments
INTERSECT
SELECT user_id FROM game_logs;
```

---

## 6. `EXCEPT` — 차집합

**왼쪽에는 있지만 오른쪽에는 없는 행.** (= 왼쪽 − 오른쪽)

```sql
-- 가입은 했지만 결제는 한 번도 안 한 유저
SELECT id FROM users
EXCEPT
SELECT user_id FROM payments;
```

```sql
-- 전체 길드 이름 - 가입자 있는 길드 이름 = 가입자 0명인 길드
SELECT guild_name FROM guilds
EXCEPT
SELECT g.guild_name
FROM guilds g
JOIN users u ON u.guild_id = g.id;
```

> 💡 다른 DB에선 `MINUS`라고 부르기도 하지만(Oracle), PostgreSQL은 **`EXCEPT`** 가 표준.

---

## 7. `ALL` 변형

`INTERSECT`와 `EXCEPT`에도 `ALL` 버전이 있다 — 중복을 그대로 유지.

```sql
-- 일반: 중복 제거
SELECT name FROM a INTERSECT SELECT name FROM b;
SELECT name FROM a EXCEPT    SELECT name FROM b;

-- ALL: 중복 유지
SELECT name FROM a INTERSECT ALL SELECT name FROM b;
SELECT name FROM a EXCEPT    ALL SELECT name FROM b;
```

`INTERSECT ALL`의 의미: "양쪽에 `'김철수'`가 각각 3번, 2번 있으면 결과에 2번."
`EXCEPT ALL`: "왼쪽에 3번, 오른쪽에 1번 있으면 결과에 2번."

실무에선 `INTERSECT ALL`/`EXCEPT ALL`은 거의 안 쓴다. 알기만 하자.

---

## 8. `ORDER BY` / `LIMIT`은 **마지막에 한 번만**

각 SELECT에 따로 `ORDER BY`를 붙일 수 없다. **전체 결과에 적용**되도록 마지막에 한 번.

```sql
-- ❌ 각각 ORDER BY는 안 됨
SELECT name FROM users WHERE guild_id = 1 ORDER BY name
UNION
SELECT name FROM users WHERE guild_id = 2 ORDER BY name;

-- ✅ 마지막에 한 번
SELECT name FROM users WHERE guild_id = 1
UNION
SELECT name FROM users WHERE guild_id = 2
ORDER BY name
LIMIT 10;
```

`ORDER BY`의 기준은 **첫 번째 SELECT의 컬럼명**이나 위치 번호(`ORDER BY 1`)로.

### 컬럼 이름이 다를 때

```sql
-- 첫 SELECT의 컬럼명이 결과의 이름이 된다
SELECT u.name AS person FROM users u
UNION
SELECT a.name FROM admins a
ORDER BY person;   -- ✅
```

---

## 9. 우선순위

세 연산자가 섞이면 우선순위는: **`INTERSECT` > `UNION` ≈ `EXCEPT`** (좌→우).

헷갈리면 **괄호로 명확히**.

```sql
-- 의도: "(A UNION B) EXCEPT C"
( SELECT name FROM users WHERE level >= 40
  UNION
  SELECT name FROM admins )
EXCEPT
SELECT name FROM banned_users;
```

각 SELECT를 괄호로 감싸도 OK:

```sql
(SELECT name FROM users WHERE level >= 40)
UNION
(SELECT name FROM admins);
```

## 10. 집합 연산 vs JOIN — 언제 무엇?

같은 결과를 양쪽으로 만들 수 있는 경우가 많다. 감각만 잡자.

| 하고 싶은 것 | 추천 |
|--------------|------|
| 두 테이블의 행을 **한 목록으로 합쳐서** 보고 싶다 | `UNION` / `UNION ALL` |
| 한 행에 **양쪽 컬럼을 펼쳐서** 보고 싶다 | `JOIN` |
| "A에는 있고 B에는 없는" 키를 찾고 싶다 | `EXCEPT` 또는 `LEFT JOIN ... WHERE b.x IS NULL` |
| "A와 B 양쪽에 모두 있는" 키를 찾고 싶다 | `INTERSECT` 또는 `INNER JOIN` |

### `EXCEPT` vs `LEFT JOIN ... IS NULL` (자주 비교됨)

```sql
-- EXCEPT 방식
SELECT id FROM users
EXCEPT
SELECT user_id FROM payments;

-- LEFT JOIN 방식 (anti-join)
SELECT u.id
FROM users u
LEFT JOIN payments p  ON p.user_id = u.id
WHERE p.user_id IS NULL;
```

결과는 보통 같다. 다만:
- `EXCEPT`는 **중복 제거가 자동**
- `LEFT JOIN` 방식은 **추가 컬럼**(payments의 다른 정보)을 같이 가져올 수 있어 유연
- 실무 쿼리에선 LEFT JOIN 방식이 좀 더 흔하다

## 11. 실습

```sql
-- 1) 길드 1 또는 2 소속 유저 이름 (중복 제거)
SELECT name FROM users WHERE guild_id = 1
UNION
SELECT name FROM users WHERE guild_id = 2
ORDER BY name;

-- 2) UNION ALL (중복 그대로) — 차이 비교
SELECT name FROM users WHERE level >= 30
UNION ALL
SELECT name FROM users WHERE level >= 40;
-- 30+ 와 40+ 양쪽에 들어가는 이름은 두 번 나옴

-- 3) 가입자 0명인 길드 (차집합 방식)
SELECT guild_name FROM guilds
EXCEPT
SELECT g.guild_name FROM guilds g JOIN users u ON u.guild_id = g.id;

-- 4) 동일 결과를 LEFT JOIN으로
SELECT g.guild_name
FROM guilds g
LEFT JOIN users u  ON u.guild_id = g.id
WHERE u.id IS NULL;

-- 5) 아이템을 가진 유저 ID와 길드 가입 유저 ID의 교집합
SELECT user_id FROM items
INTERSECT
SELECT id FROM users WHERE guild_id IS NOT NULL;

-- 6) "지금 모든 알림 받을 사람" 목록 만들기
SELECT name, 'admin' AS role FROM admins
UNION ALL
SELECT name, 'vip'   AS role FROM users WHERE level >= 50
ORDER BY role, name;
-- → 한 결과에 종류(role)를 함께 표시하는 흔한 패턴
```

> 💡 마지막 예시처럼 **고정 문자열 컬럼(`'admin'`)을 함께 SELECT**해서 결과의 출처를 구분하는 트릭은 자주 쓴다.

---

## 12. 이번 강의에서 기억할 것

1. **`UNION` / `UNION ALL` / `INTERSECT` / `EXCEPT`** — 세로 결합 4종.
2. **컬럼 수와 타입이 양쪽 동일**해야 함. 결과 컬럼명은 **첫 번째 SELECT** 기준.
3. **`UNION`은 중복 제거(느림), `UNION ALL`은 그대로(빠름).** 중복 가능성 없으면 `UNION ALL`.
4. **`INTERSECT`는 교집합, `EXCEPT`는 차집합.**
5. **`ORDER BY` / `LIMIT`은 마지막 SELECT 뒤에 한 번만.**
6. `EXCEPT` ↔ `LEFT JOIN ... IS NULL`은 호환 가능한 패턴.
7. **출처 구분용 고정 문자열 컬럼**(`'admin' AS role`)은 UNION 결과에서 유용.
