# 1강 — 데이터베이스란 무엇인가?

> **목표:** "DB가 도대체 왜 필요한가?"에 대한 감을 잡는다.
> **선수 지식:** 없음. C++로 파일 입출력(`fstream`) 정도 해본 경험이면 충분.

---

## 1. 일단, 게임을 만든다고 상상해보자

언리얼로 RPG 게임을 만들고 있다고 해보자. 유저 데이터를 저장해야 한다.

- 유저 이름
- 레벨
- 골드
- 보유 아이템 목록

C++로 짠다면 보통 이렇게 시작할 거다.

```cpp
struct FUserData
{
    FString Name;
    int32   Level;
    int32   Gold;
    TArray<FItem> Items;
};

TArray<FUserData> AllUsers;  // 모든 유저를 메모리에 보관
```

그런데 게임을 끄면? **메모리는 다 날아간다.** 그래서 파일로 저장한다.

```cpp
// 게임 종료 시
SaveToFile("users.dat", AllUsers);

// 게임 시작 시
LoadFromFile("users.dat", AllUsers);
```

여기까지는 별 문제 없어 보인다. 그런데…

---

## 2. 파일 저장의 한계

게임이 잘 돼서 유저가 **100만 명**이 됐다고 해보자. 그러면 이런 문제가 터진다.

### 문제 1) "레벨 50 이상인 유저만 보고 싶다"

파일을 통째로 다 읽어서, 100만 명을 전부 메모리에 올린 다음, `for`문 돌려서 필터링해야 한다.
👉 메모리도, 시간도 낭비.

### 문제 2) 여러 사람이 동시에 접근하면?

게임 서버 A와 게임 서버 B가 같은 파일을 동시에 읽고 쓰면 데이터가 깨진다.
파일 잠금(lock)을 직접 구현해야 한다. 끔찍하다.

### 문제 3) 도중에 프로그램이 죽으면?

골드 차감하다가 서버가 죽었다. 골드는 빠졌는데 아이템은 못 받았다. → **데이터 정합성 깨짐.**

### 문제 4) 데이터끼리 연결이 복잡해진다

- 유저는 길드에 속한다
- 길드는 여러 유저를 가진다
- 유저는 여러 아이템을 가진다
- 아이템은 강화 이력이 있다…

이걸 다 파일로? 직접? 생각만 해도 머리가 아프다.

---

## 3. 그래서 데이터베이스(DB)가 등장한다

**데이터베이스(Database)** 는 위 문제들을 전부 해결해주는, **데이터를 저장하고 관리하는 전문 소프트웨어**다.

비유하자면:

| 직접 파일 관리 | 데이터베이스 사용 |
|---|---|
| 메모장에 직접 적어서 보관 | 도서관 사서에게 맡김 |
| 내가 다 찾고, 다 정리해야 함 | "레벨 50 이상 유저 줘"라고 요청만 하면 됨 |
| 동시 접근, 충돌 직접 처리 | DB가 알아서 처리 |
| 죽으면 데이터 깨짐 | **트랜잭션**으로 안전 보장 |

---

## 4. DBMS — DB를 관리하는 프로그램

용어 하나 짚고 가자.

- **DB (Database):** 데이터 그 자체. 실제 저장된 정보들.
- **DBMS (Database Management System):** DB를 관리해주는 **소프트웨어**.

우리가 이번 강의에서 배울 **PostgreSQL이 바로 DBMS**다.
같은 부류로 MySQL, Oracle, SQL Server, SQLite 등이 있다.

> 💡 C++로 비유하면, DB는 "데이터", DBMS는 "데이터를 다뤄주는 라이브러리/엔진"이라고 보면 된다.

---

## 5. DB와 대화하는 언어 — SQL

C++로 변수에 값을 넣을 때 `int x = 10;` 이라고 쓰듯이,
DB에게 명령을 내릴 때도 **전용 언어**가 있다. 그게 바로 **SQL** (Structured Query Language, "시퀄" 또는 "에스큐엘"로 읽음).

예시 한번 미리 구경:

```sql
-- 레벨 50 이상인 유저 이름과 골드를 조회
SELECT name, gold
FROM users
WHERE level >= 50;
```

읽어보면 거의 영어 문장이다.
> "users에서 / name, gold를 / 가져와 / 조건은 level이 50 이상"

이게 SQL의 매력이다. **사람이 읽기 쉽게 설계됐다.**

---

## 6. 왜 하필 PostgreSQL인가?

세상엔 DBMS가 많은데 왜 PostgreSQL을 배우나?

- **무료, 오픈소스** — 라이선스 걱정 없음
- **표준 SQL을 잘 지킨다** — 여기서 배운 거 다른 DB에서도 거의 그대로 통함
- **기능이 풍부하다** — 게임 서버, 웹, 데이터 분석 등 거의 모든 분야에서 사용
- **업계에서 인기 상승 중** — 신입이 익혀두면 가성비 최고

---

## 7. 이번 강의에서 기억할 것

1. DB는 **데이터를 안전하고 효율적으로 관리해주는 도구**다.
2. **DB(데이터) ≠ DBMS(관리 프로그램)**. PostgreSQL은 DBMS.
3. DB와 대화할 때는 **SQL**이라는 언어를 쓴다.
4. 파일 저장과 달리 DB는 **검색, 동시 접근, 정합성** 문제를 다 해결해준다.
