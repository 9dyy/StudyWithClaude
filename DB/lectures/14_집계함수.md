# 14강 — 집계 함수 자세히

> **목표:** `COUNT`, `SUM`, `AVG`, `MAX`, `MIN`을 정확히 이해하고, NULL이 끼치는 영향까지 안다.
> **선수 지식:** 13강 (`GROUP BY`)

---

## 1. 집계 함수란

**여러 행의 값을 받아 하나의 값으로 요약하는** 함수. 13강에서 봤지만 이번엔 디테일을 판다.

| 함수 | 하는 일 | 대상 타입 |
|------|---------|----------|
| `COUNT` | 개수 세기 | 아무 타입 |
| `SUM` | 합계 | 숫자 |
| `AVG` | 평균 | 숫자 |
| `MAX` | 최댓값 | 숫자, 문자, 날짜 |
| `MIN` | 최솟값 | 숫자, 문자, 날짜 |

---

## 2. `COUNT` — 가장 자주, 가장 헷갈리는

세 가지 형태가 있고 **결과가 다르다.**

```sql
SELECT
    COUNT(*)             AS 전체행수,      -- 모든 행
    COUNT(guild_id)      AS 길드있는수,    -- guild_id가 NULL이 아닌 행
    COUNT(DISTINCT guild_id) AS 길드종류수  -- 중복 제거한 guild_id 개수
FROM users;
```

7강 데이터 기준 결과 예시:

```text
 전체행수 | 길드있는수 | 길드종류수
----------+------------+------------
        6 |          5 |          3
```

### 결정적 차이

| 형태 | 세는 것 | NULL 처리 |
|------|---------|----------|
| `COUNT(*)` | **모든 행** | NULL 포함 |
| `COUNT(열)` | 그 열이 **NULL이 아닌** 행 | NULL 제외 |
| `COUNT(DISTINCT 열)` | 그 열의 **고유값** 개수 | NULL 제외 |

홍길동은 `guild_id`가 NULL이라, `COUNT(*)`엔 포함되지만 `COUNT(guild_id)`엔 빠진다. **이 차이가 실무에서 버그의 단골 원인.**

> 💡 "행이 몇 개야?"는 거의 항상 **`COUNT(*)`**. 특정 열의 "값이 있는 것만 세고 싶다"면 `COUNT(열)`.

---

## 3. `SUM` — 합계

```sql
SELECT SUM(coin) AS 총코인 FROM users;
SELECT SUM(quantity) AS 총아이템수 FROM items;
```

### NULL은 무시된다

`SUM`은 NULL을 **그냥 건너뛴다** (0으로 치지 않음). 보통 원하는 대로 동작.

### ⚠️ 모든 값이 NULL이거나 행이 없으면?

```sql
SELECT SUM(coin) FROM users WHERE level > 9999;  -- 해당 행 없음
```

결과는 **0이 아니라 NULL**이다! 이게 함정. 0을 원하면 `COALESCE`로 감싼다 (32강).

```sql
SELECT COALESCE(SUM(coin), 0) AS 총코인 FROM users WHERE level > 9999;
```

---

## 4. `AVG` — 평균

```sql
SELECT AVG(level) AS 평균레벨 FROM users;
```

결과가 소수로 나온다 (`22.5000...`). 보기 좋게 반올림:

```sql
SELECT ROUND(AVG(level), 1) AS 평균레벨 FROM users;   -- 22.5
```

### ⚠️ AVG도 NULL을 무시한다 — 분모 주의

`AVG`는 **NULL인 행을 분모에서 뺀다.**

예를 들어 `score` 열에 값 3개, NULL 2개가 있으면:
- `AVG(score)` = (값들의 합) ÷ **3** (NULL 2개 제외)
- "NULL을 0으로 치고 5로 나눈" 평균과 **다르다.**

이게 의도와 다를 수 있으니, NULL을 0으로 보고 싶으면 미리 처리:

```sql
SELECT AVG(COALESCE(score, 0)) FROM ...;   -- NULL을 0으로 쳐서 평균
```

---

## 5. `MAX` / `MIN` — 최대·최소

숫자만이 아니라 **문자열, 날짜**에도 쓸 수 있다.

```sql
SELECT MAX(level), MIN(level) FROM users;       -- 숫자
SELECT MAX(name), MIN(name) FROM users;          -- 문자열 (사전순)
SELECT MAX(created_at) AS 최근가입 FROM users;   -- 날짜 (가장 최근)
SELECT MIN(created_at) AS 최초가입 FROM users;   -- 날짜 (가장 오래된)
```

- 문자열의 `MAX`/`MIN`은 **사전 순서** 기준.
- 날짜의 `MAX`는 **가장 미래**, `MIN`은 **가장 과거**.
- `MAX`/`MIN`도 NULL은 무시한다.

---

## 6. 여러 집계를 한 줄에

```sql
SELECT
    COUNT(*)          AS 유저수,
    ROUND(AVG(level), 1) AS 평균레벨,
    MAX(level)        AS 최고레벨,
    MIN(level)        AS 최저레벨,
    SUM(coin)         AS 총코인
FROM users;
```

대시보드용 통계 한 방. 실무에서 정말 자주 쓰는 패턴.

## 7. `GROUP BY`와 결합 (복습 + 심화)

13강에서 본 그룹별 집계. 이제 함수를 풍부하게.

```sql
SELECT
    guild_id,
    COUNT(*)             AS 인원,
    ROUND(AVG(level), 1) AS 평균레벨,
    MAX(level)           AS 최고레벨,
    SUM(coin)            AS 총코인
FROM users
GROUP BY guild_id
ORDER BY 평균레벨 DESC;
```

---

## 8. `FILTER` — 조건부 집계 (PostgreSQL 강점)

"전체 인원 중 레벨 30 이상은 몇 명?"을 **한 줄에** 같이 구할 수 있다.

```sql
SELECT
    COUNT(*)                          AS 전체,
    COUNT(*) FILTER (WHERE level >= 30) AS 고렙,
    COUNT(*) FILTER (WHERE coin > 1000) AS 부자
FROM users;
```

결과:

```text
 전체 | 고렙 | 부자
------+------+------
    6 |    3 |    2
```

`FILTER (WHERE 조건)`은 그 집계 함수에만 적용되는 조건. 여러 조건별 통계를 한 쿼리로 뽑을 때 매우 유용. (다른 DB에선 `CASE WHEN` 트릭을 쓰는데 PG는 깔끔하게 `FILTER` 제공.)

## 9. 흔한 실수 모음

### ① 집계 함수와 일반 열을 그냥 섞기

```sql
SELECT name, COUNT(*) FROM users;   -- ❌ (GROUP BY 없이)
```
→ 13강 규칙 위반. `name`을 어느 행 걸 보여줄지 모름.

### ② `WHERE`에 집계 함수

```sql
SELECT guild_id FROM users WHERE COUNT(*) > 1 GROUP BY guild_id;  -- ❌
```
→ `HAVING COUNT(*) > 1`로.

### ③ `SUM`/`AVG` 결과가 NULL인 걸 깜빡

→ 행이 없으면 `SUM`은 NULL. `COALESCE(SUM(x), 0)`로 방어.

### ④ `COUNT(열)`로 전체 행을 세려다 NULL 누락

→ 전체 행은 `COUNT(*)`.

## 10. 실습

```sql
-- 1) 전체 통계 한 방
SELECT
    COUNT(*) AS 유저수,
    ROUND(AVG(level), 1) AS 평균레벨,
    MAX(level) AS 최고,
    MIN(level) AS 최저,
    SUM(coin) AS 총코인
FROM users;

-- 2) COUNT 세 형태 비교
SELECT
    COUNT(*) AS 전체,
    COUNT(guild_id) AS 길드있음,
    COUNT(DISTINCT guild_id) AS 길드종류
FROM users;

-- 3) 길드별 통계
SELECT guild_id, COUNT(*) AS 인원, ROUND(AVG(level),1) AS 평균레벨
FROM users
GROUP BY guild_id
ORDER BY 인원 DESC;

-- 4) 조건부 집계 (FILTER)
SELECT
    COUNT(*) AS 전체,
    COUNT(*) FILTER (WHERE level >= 30) AS 고렙,
    ROUND(AVG(level) FILTER (WHERE coin > 1000), 1) AS 부자평균레벨
FROM users;

-- 5) 아이템: 유저별 총 보유 수량
SELECT user_id, SUM(quantity) AS 총수량
FROM items
GROUP BY user_id
ORDER BY 총수량 DESC;

-- 6) NULL 방어
SELECT COALESCE(SUM(coin), 0) AS 총코인
FROM users WHERE level > 9999;   -- 0이 나옴 (NULL 아님)
```

## 11. 이번 강의에서 기억할 것

1. **`COUNT(*)`는 모든 행, `COUNT(열)`은 NULL 제외, `COUNT(DISTINCT 열)`은 고유값.**
2. **`SUM`/`AVG`/`MAX`/`MIN`은 NULL을 무시**한다.
3. **행이 없으면 `SUM`/`AVG`는 0이 아니라 NULL** → `COALESCE`로 방어.
4. `AVG`는 NULL을 분모에서 빼므로, 의도와 다를 수 있다.
5. **`FILTER (WHERE ...)`** 로 조건부 집계를 한 쿼리에 (PG 강점).

## 12. 다음 강의 예고

**15강 — `DISTINCT`로 중복 제거**
조회 결과에서 중복을 없애는 법. `DISTINCT`와 `DISTINCT ON`, 그리고 `GROUP BY`로 중복 제거하는 것과 뭐가 다른지까지 정리한다. 이걸로 **3단계(SELECT 시리즈)가 끝난다.**
