# 운영체제 강의 계획서

> **수강생:** 언리얼 C++ 클라이언트 개발 신입 (OS 한 번 배웠지만 휘발됨)
> **목표:** 이직 면접에서 "운영체제 좀 아는 클라 개발자"로 보일 수준까지.
> **분량:** 본강 15강 + 보너스 1강

---

## 이 강의가 지향하는 것

대학교 OS 교재처럼 "OS의 5대 기능을 서술하시오" 같은 걸 외우는 강의가 아니다.
**언리얼 C++로 게임을 만드는 사람이 실제로 겪는 상황**과 OS 개념을 1:1로 붙여서 설명한다.

예시:
- "왜 게임 스레드에서 렌더 스레드 데이터를 직접 못 만지지?" → 동기화 / Race Condition
- "왜 `TArray`를 `for`로 도는 게 `TMap`보다 빠르지?" → 캐시 지역성
- "왜 텍스처를 비동기 로딩하지?" → 블로킹 I/O와 메인 스레드
- "왜 메모리는 충분한데 게임이 끊기지?" → 페이지 폴트, 가상 메모리

각 강의는 **"이걸 모르면 면접에서 답 못하는 것"** 과 **"이걸 모르면 코드 짤 때 헛발질하는 것"** 을 모두 다룬다.

---

## 학습 방식

- 각 강의는 별도 `.md` 파일로 `lectures/` 폴더에 둔다.
- 모든 코드 예제는 **C++ 표준 또는 언리얼 API**로 작성한다. (Python, Java X)
- 강의 끝에는 **면접 질문 3~5개**와 **간단한 셀프 체크**를 둔다.
- 한 강의 분량: 읽는 데 15~25분.

1. 이 계획서를 보고 **순서를 바꾸거나 빼고 싶은 강의**가 있는지 먼저 의견 주기

3. 각 강의 끝에서 "이해 됐는지" 짧게 확인 후 다음 강의로 넘어감
4. 막히는 강의는 별도 보충 강의를 추가 (예: `09-1_메모리배리어_보충.md`)

## 참고: 이 강의가 "다루지 않는" 것

이직 면접 + 클라 개발자 일상에 거의 안 나오는 주제는 의도적으로 뺐다.

- 디스크 스케줄링 알고리즘 (SCAN, C-SCAN 등) — 거의 안 물어봄
- 분산 시스템 / 분산 합의 — 클라 면접 범위 밖
- 리눅스 커널 코드 분석 — Windows 게임 클라가 주력이라
- 보안 (ASLR, DEP) 디테일 — 필요하면 보너스에서 한 줄

원한다면 추가/제외 가능. 알려주면 반영한다.
